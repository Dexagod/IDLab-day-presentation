Main message

Point1

  • making data available on the Web in a flexible way, we want to maximize the possibility to integrate with our data

Point2

  • protecting data on the Web, try to automate user behavior and wishes, business processes, ...

Point3

  • provide overview of data on the Web, and keep track of exchanges. We have receipts when exchanging money for services, we should do the same when exchanging data for services

Point4

  • putting these things into action, we can enable agents to flexibly exchange parts of their internal knowledge graphs according to their interfaces and policies, and control the interactions in hindsight using our receipts to detect misuse. This allows much more flexible integration of data in the Web ecosystem
